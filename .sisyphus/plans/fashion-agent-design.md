## 第一部分：产品定义

### 1.1 产品定位

**一句话描述**：为服装带货博主量身定制的AI脚本创作智能体——它懂你的货、懂你的风格、懂爆款逻辑。

**核心价值主张**：
- 输入一个新品（如"白色直筒裤"），智能体自动生成一整套符合你个人风格的带货脚本
- 不是通用AI文案，而是"你的专属编剧"

**与市面产品的核心差异**：

| 维度 | 市面产品（爆单王AI/陌言AI/蝉妈妈） | 本产品 |
|------|--------------------------------------|--------|
| 知识库 | 无个人知识库，通用模板 | 私有知识库：你的产品、你的历史脚本、你的风格 |
| 风格 | 通用AI风格，输出同质化 | 基于你的历史文案学习个人调性 |
| 爆款分析 | 简单抄文案/抓热榜 | 拆解爆款视频底层逻辑框架 |
| 脚本结构 | 无固定结构，自由生成 | 强制按"钩子→内容→干货→总结"输出 |
| 数据 | SaaS，数据在别人服务器 | 私有部署，数据完全自主 |

### 1.2 目标用户

| 维度 | 描述 |
|------|------|
| 主要用户 | 有一定粉丝基础的服装带货博主（抖音/小红书/快手） |
| 次要用户 | 服装电商团队的运营/编导人员 |
| 典型场景 | 新品上架前需要快速出脚本，同时保证内容和平时风格一致 |

### 1.3 核心使用流程

```
首次使用（一次性）
    │
    ├─ Step 1：上传产品资料（SKU表、产品描述、面料/尺码/价格等）
    ├─ Step 2：导入历史脚本/文案（往期所有作品的文字稿）
    └─ Step 3：设定风格标签（可选，如"亲和力""专业感""接地气"等）

日常使用
    │
    ├─ Step A：输入新品名称（如"白色高腰直筒裤"）
    ├─ Step B：可选——输入1-N条参考爆款视频链接
    ├─ Step C：智能体自动完成：
    │     ├─ 从知识库检索该新品的产品信息
    │     ├─ 从知识库检索最匹配的历史脚本（风格参考）
    │     ├─ 分析爆款视频（如有），提取逻辑框架
    │     └─ 按固定结构生成新品带货脚本
    └─ Step D：用户查看/微调脚本 → 导出使用
```

### 2.2 模块详细设计

#### 模块一：知识库管理

**目的**：让智能体"懂你的货"和"懂你的风格"

| 子功能 | 描述 | 输入 | 输出 |

| 产品资料上传 | 批量导入产品SKU信息 | Excel/CSV/手动填写 | 结构化产品数据库 |
| 历史脚本导入 | 导入往期所有作品的文字稿 | TXT/Word/粘贴文本 | 带标签的脚本库 |
| 风格档案 | 系统自动分析+人工标注风格标签 | 历史脚本 + 用户标注 | 风格向量/标签集 |
| 知识检索 | 语义搜索知识库内容 | 自然语言查询 | 相关文档片段 |

#### 模块二：爆款视频分析

**目的**：让智能体"懂爆款逻辑"

| 子功能 | 描述 | 输入 | 输出 |
|--------|------|------|------|
| 视频链接导入 | 支持抖音/小红书/快手/B站链接 | 视频URL | 原始视频文件 |
| 自动转录 | 提取视频中的口播文字 | 视频文件 | 完整文稿 |
| 场景切分 | 按镜头/场景自动分段 | 视频文件 | 分段时间轴 |
| 逻辑框架提取 | AI分析视频的说服结构 | 文稿+场景信息 | 结构化框架 |
| 爆款评分 | 评估各段的"爆款潜力" | 框架+文稿 | 评分报告 |

**爆款逻辑框架输出格式**：

#### 模块三：脚本工坊（核心功能）

**目的**：生成符合个人风格+固定结构的新品带货脚本

**脚本生成流程**：
```
用户输入
    │
    ├─ 新品名称/选择已有产品
    ├─ 可选：选择1-N条参考爆款视频
    └─ 可选：补充本次特别要求（如"突出性价比"）
    │
    ▼
智能体内部处理
    │
    ├─ ① 从产品库检索新品详细信息
    ├─ ② 从历史脚本库检索最相似的3-5篇脚本（风格参考）
    ├─ ③ 如有参考爆款，加载爆款逻辑框架
    ├─ ④ 构造Prompt：
    │     ├─ 系统角色：你是一位服装带货脚本编剧
    │     ├─ 风格锚定：以下是该博主的历史脚本样本
    │     ├─ 产品信息：新品详细资料
    │     ├─ 爆款参考：拆解的爆款逻辑框架
    │     └─ 输出结构：强制按四段式输出
    └─ ⑤ 大模型生成脚本
    │
    ▼
输出固定结构脚本
```

**固定脚本结构模板**：

```
═══════════════════════════════════════
【新品带货脚本】白色高腰直筒裤
═══════════════════════════════════════

🎬 第一步：视觉钩子（0-3秒）
─────────────────────────────────────
[画面]：......
[口播]：......
[目的]：3秒内抓住注意力，制造悬念/痛点/好奇

📌 第二步：内容输出（3-25秒）
─────────────────────────────────────
[画面]：......
[口播]：......
[目的]：产品核心卖点展示，建立购买欲望

💡 第三步：干货分享（25-38秒）
─────────────────────────────────────
[画面]：......
[口播]：......
[目的]：提供实用价值（穿搭技巧/避坑指南），
       建立专业信任感

🎯 第四步：总结+行动号召（38-45秒）
─────────────────────────────────────
[画面]：......
[口播]：......
[目的]：总结核心卖点 + 引导下单/关注

═══════════════════════════════════════
预估时长：XX秒 | 字数：XX字
参考爆款：[视频标题] 的钩子技巧
风格匹配度：85%（基于你近10篇脚本）
═══════════════════════════════════════
```

**脚本编辑器功能**：
- 在线编辑修改生成结果
- 一键重新生成某个段落（如只对"钩子"不满意，重新生成钩子部分）
- 保存到历史脚本库
- 导出为TXT/Word
- 复制到剪贴板

### 3.2 技术选型

| 层级 | 组件 | 推荐方案 | 备选方案 | 选型理由 |
|------|------|---------|---------|---------|
| **AI编排** | 工作流引擎 | **Dify** | LangFlow | 开箱即用、中文生态强、RAG+Agent+工作流一体 |
| **大模型** | LLM | **DeepSeek-V3** | Qwen2.5 / GPT-4o | 性价比高、中文能力强、支持私有部署 |
| **知识库** | 向量数据库 | **Qdrant** | Weaviate / Milvus | Dify原生支持、轻量、性能好 |

| **文件存储** | 对象存储 | **MinIO** | AWS S3 | 私有部署兼容S3 API |
| **视频下载** | 下载器 | **yt-dlp** | — | 支持1000+平台、活跃维护 |
| **语音转录** | ASR | **faster-whisper** | Whisper / FunASR | 比OpenAI Whisper快4倍、显存占用低 |

### 3.4 Dify工作流设计

在Dify中需要创建3个核心工作流：

**工作流1：知识库录入（dataset-ingestion）**
```
开始节点
  → 文档解析节点（PDF/Word/TXT/CSV解析）
  → 文本分块节点（Chunk策略：按段落+重叠）
  → Embedding节点（调用Embedding模型）
  → 存储节点（写入Qdrant）
  → 元数据存储（写入PostgreSQL）
结束节点
```

**工作流2：爆款视频分析（video-analysis）**
```
开始节点（接收视频URL）
  → HTTP请求节点（调用视频处理微服务）
      → 返回：转录文稿 + 场景切分 + 关键帧
  → LLM节点（分析爆款逻辑框架）
      → System Prompt：爆款视频分析师角色
      → 输入：转录+场景+关键帧描述
      → 输出：结构化JSON
  → 知识库存储节点
  → 结束节点（返回分析结果）
```

**工作流3：脚本生成（script-generation）**
```
开始节点（接收：新品名称 + 爆款参考ID + 特别要求）
  → 知识库检索节点①（产品信息）
  → 知识库检索节点②（相似历史脚本，Top 5）
  → 条件判断节点（是否有爆款参考）
      → 是：数据库查询节点（加载爆款分析）
      → 否：跳过
  → LLM节点（主生成）
      → 使用完整Prompt模板
      → Temperature: 0.7（平衡创意和一致性）
  → LLM节点（质量检查）
      → 验证4段结构完整性
      → 评估风格匹配度
  → 结束节点（返回脚本 + 评分）
```

### 3.5 视频处理微服务设计

独立Python微服务，负责视频下载、转录、场景切分：

```
视频处理微服务（FastAPI）
├── POST /api/v1/video/process
│   输入：{"video_url": "https://..."}
│   输出：{
│     "transcript": "完整转录文本",
│     "scenes": [{"start": 0, "end": 3.2, "type": "hook"}, ...],
│     "keyframes": [{"timestamp": 1.5, "image_path": "..."}, ...],
│     "duration": 45.2
│   }
│
├── 内部流水线：
│   1. yt-dlp下载视频 → /tmp/videos/
│   2. FFmpeg提取音频 → /tmp/audio/
│   3. faster-whisper转录 → transcript.json
│   4. TransNetV2场景切分 → scenes.json
│   5. FFmpeg关键帧提取 → /tmp/keyframes/
│   6. 清理临时文件
│
└── 技术细节：
    - faster-whisper模型：large-v3（中文效果最好）
    - TransNetV2：PyTorch推理，CPU即可
    - 并发：Celery任务队列，最多3个并发处理
    - 超时：单视频处理限时5分钟
```

## 第六部分：风险与应对

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|---------|
